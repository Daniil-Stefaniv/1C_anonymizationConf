## Причина 1. Ложная XML-ошибка из-за BOM

Проверка реальных файлов из отчета показала, что проблемные XML начинаются с UTF-8 BOM:

```text
EF BB BF 3C 3F 78 6D 6C ...
```

Это обычное начало файла:

```text
﻿<?xml version="1.0" ...
```

В старой версии GUI BOM попадал в строку, передаваемую в `XmlReader` через `StringReader`. Из-за этого валидатор видел символ перед корневым XML-элементом и ошибочно сообщал:

```text
Недопустимые данные на корневом уровне., строка 1, позиция 1.
```

Файлы с такой ошибкой не были записаны, потому что программа останавливала запись при неуспешной XML-валидации.

## Дополнительный риск. Были затронуты не-1С файлы

Отчет показал успешные изменения в служебных файлах рядом с выгрузкой:

- `clean_1c_comments.py`

Причина та же: расширенный режим сканировал неизвестные текстовые файлы и применял XML-очистку к содержимому, которое не является файлом выгрузки 1С. В результате исходники очистителя были повреждены. Оба файла восстановлены.

## Внесенные исправления

Исправлен `Clean1CCommentsGui.cs` и пересобран `Clean1CCommentsGui.exe`.

Что изменено:

- UTF-8/UTF-16 BOM удаляется из рабочей строки перед XML-валидацией.
- Исходный BOM сохраняется при обратной записи файла.
- JSON-отчет GUI теперь пишется в UTF-8 with BOM, чтобы русские пути корректно открывались в стандартных Windows-инструментах.
- Бинарные форматы пропускаются по расширению и сигнатурам: ZIP, PNG, JPG, GIF, BMP, PDF, EXE/MZ и др.
- Пропущенные бинарные файлы больше не считаются ошибками обработки.
- Расширенное сканирование неизвестных расширений стало безопасным: файл меняется только если его содержимое похоже на XML выгрузки 1С.
- Удаление BSL-комментариев применяется только к `.bsl` и `.os`.
- В отчет добавлено поле `skip_reason`.
- Восстановлены и расширены self-tests.

Также восстановлен `clean_1c_comments.py`:

- возвращен regex удаления XML-комментариев;
- восстановлен self-test;
- добавлена защита от бинарных файлов;
- добавлено корректное сохранение BOM;
- режим `--all-text` стал консервативным.

## Рекомендации по дальнейшему запуску

1. Использовать новую версию `Clean1CCommentsGui.exe`.
2. Сначала запускать `Проверить + XML-валидация`.
3. Если ошибок нет, запускать `Очистить + backup`.
4. Расширенную проверку неизвестных текстовых файлов включать только при необходимости.
5. После реальной очистки обязательно выполнить загрузку конфигурации в пустую базу 1С или проверку штатной утилитой 1С.
